# Use a Knowledge-gated structured Lens over PALI

Normal play will manipulate a bounded typed PALI document through a
Knowledge-gated Lens: forms for State and readable Clauses for Behavior and
Law. Raw PALI remains the normalized serialized language and a developer or
eventual root-level view, but it is not the default player interface. This
preserves PALI as the Universe's executable grammar while preventing malformed
source, making mouse interaction complete, and allowing the interface itself
to deepen as Knowledge grows. Reach is validated independently: Entity Reach
is available early, while the existing shared-Prototype mechanism is retained
as developer/testable Prototype Reach and withheld from normal Knowledge until
late progression.

## Consequences

The parser must produce a typed document that can be projected, changed, and
deterministically formatted before the compiler emits bytecode. UI controls
may offer only known and type-compatible operations; host implementation fields
never enter the document. PALI names resolve through a bounded Lexicon carrying
stable concept identity, semantic type, Facet, constraints, and available
notations; the Lens does not special-case property-name strings. Entity and
broader Patches are sparse overlays on stable semantic node identities rather
than copied complete definitions, so unrelated meaning continues resolving
through broader Provenance. Existing transactional compilation, fixed
capacities, and last-known-valid behavior remain mandatory.
