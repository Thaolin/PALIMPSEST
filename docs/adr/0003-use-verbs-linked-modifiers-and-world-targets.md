# Use Verbs, linked Modifiers, and world Targets for Expressions

**Status:** accepted

The accepted `Verb[Noun]` model made collectible Nouns select subjects without
changing the operation enough to create the promised build depth. Expressions
will instead configure one authored Verb with compatible linked Modifiers. On
Invocation, the player selects an actual Target from Chronicle state; Targets
are not Words, and their matter, scale, identity, agency, and other world facts
determine whether the configured magic can affect them.

This preserves authored semantics while separating three player questions:
the Verb says what magic occurs, Modifiers say how it acts, and the Target says
which existing thing the player is trying to affect. Loadouts constrain breadth
through Verb slots and depth through link capacity plus shared Load. The
accepted Slice 5 `Fly[Stone]` and `Fly[Bell]` runtime remains predecessor
behavior until a separately authorized pressure test proves the successor fun
and a later production contract defines migration.

Link count is not a universal time surcharge. Ordinary combat and exploration
must remain responsive; substantial Chronicle-time commitment belongs to the
scale and persistence of an effect or to an explicit Modifier, with other costs
able to move into shared Load, material power, notice, instability, or
collateral.

Modifiers are reusable Codex knowledge, cannot repeat within one Expression,
and compose without order semantics. Each equipped Word attachment has fixed
authored Load independent of the Target. Targets expose constraints rather than
exact recipes, so different Expressions may satisfy the same world problem.
Most lasting Load growth comes from vulnerable Home infrastructure: losing a
Load Source reduces capacity at the next Attunement, including one for a
replacement Incarnation, instead of remotely disabling an existing Loadout.
