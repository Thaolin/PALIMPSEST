# Vision

PALIMPSEST begins as a warm, legible survival RPG and gradually reveals that
its universe is written in an editable language.

> If it exists, the player can eventually open it.

Stones, creatures, memories, diseases, civilizations, physical laws, gods, and
death can all possess inspectable source. Access deepens through five layers:

1. **State** — alter one particular apple.
2. **Behavior** — alter what that apple does when used.
3. **Inheritance** — alter its descendants.
4. **Archetype** — redefine what an apple is.
5. **Law** — alter nourishment, identity, gravity, or death.

Normal play does not expose that source as a programming exercise. Knowledge
projects it through a structured **Lens**: typed Facets for State and readable
Clauses for Behavior and Law. Raw PALI is developer machinery and a possible
late root-level view. Access Depth determines which layer can be perceived,
Concept Access determines which meanings are legible or patchable, and Reach
determines whether a Patch binds to one Entity or something broader.

Early Reach stops at one Entity. A Patch affecting every apple is Prototype
Reach and belongs to late progression even though the engine supports and
tests that scope earlier. See [`INTERACTION.md`](INTERACTION.md) for the
interaction and progression contract.

The destination is ontology design rather than a catalogue of scripted jokes.
Edits should compose through ordinary system rules: a lie with mass falls under
gravity; a fire that burns names changes which named laws can match an entity.
Unexpected results should emerge from shared semantics.

Large spatial features such as caves, villages, lakes, and ruins should come
from deterministic **Site Blueprints**, not frozen map stamps. A Site Blueprint
defines constraints, anchors, variation streams, and provenance so generated
places remain reproducible, composable, and eventually inspectable.

Player identity has three independent layers:

- **Universe**: physical state, history, entities, prototypes, and laws.
- **Knowledge**: understood language concepts, true names, and access rights.
- **Embodiment**: the creature presently inhabited and its physical needs.

The implemented milestone keeps those stores separate and exposes the first
Knowledge-gated Lens: one Entity's nourishment is Patchable while existing
Behavior is readable. Raw source is explicit developer machinery. The game
remains a finite clearing with one embodiment and no historical branching. The
visual language is an illuminated manuscript crossed with an old computer:
restrained ink and parchment, low-resolution silhouettes, firelight, and
syntax-like decorative marks whose meaning can later become literal.

The eventual save philosophy is:

> Save the recipe, the scars, and the things that acquired meaning.

Deterministic genesis supplies the recipe. Sparse materialized changes are the
scars. Later milestones may add append-only history, content-addressed law
patches, and ascended branches; this slice does not pretend those systems exist.
